Q: Why choose Web Components for this FAQ?

Web Components provide encapsulation and reusability without requiring a front-end framework.

Each faq-item is self-contained, while the parent faq-section manages their state. This keeps the code lightweight, portable, and future-proof, leveraging native browser standards.

Q: How does the accordion behavior work?

The faq-section component manages the “open” state of all its faq-item children. When a user clicks a question, the parent automatically removes the open attribute from all other items. This ensures that only one answer is visible at a time, keeping the FAQ uncluttered and easy to scan.

Animating an element’s height in CSS can be tricky to achieve smoothly. To make transitions fluid and performant, each answer uses a grid layout, animating the grid-template-rows property instead of directly changing the element’s height.

Q: How is this FAQ accessible?

ARIA attributes such as aria-expanded, aria-controls, and role are added to each faq-item to ensure screen readers announce when an item is expanded or collapsed. This makes the FAQ usable and understandable for everyone, including users navigating without visual cues.

For keyboard navigation, the inert attribute is applied to hidden answers so that any links or buttons inside them are removed from the tab order and cannot receive focus while the answer is collapsed.
